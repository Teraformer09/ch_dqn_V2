# Ch-DQN: Hierarchical Value Learning under Non-IID Partial Observability

A reference implementation of **Ch-DQN**, a hierarchical framework for stable reinforcement learning under non-IID, partially observable environments. Rather than presenting a heuristic Bellman variant, Ch-DQN structures representation learning and value updates around a central scientific thesis: **verify latent beliefs before trusting them for value propagation**.

---

## 1. Scientific Identity & Premise

In partially observable environments, standard Deep Q-Networks (DQN) suffer from representation drift and value propagation instability. In non-IID settings, this problem is exacerbated by distribution changes in observations:
$$\mathcal D_t \neq \mathcal D_{t+1}$$

**Ch-DQN** operates on the core assumption:
> **The dominant failure mode in non-IID RL is not merely observation noise—it is unstable latent belief evolution.**

To address this, the framework builds a hierarchical belief system that separates **causal predictions** (used for online policy execution) from **temporal smoothing** (used during training to verify internal consistency).

---

## 2. Mathematical Foundation

### 2.1. Environment Model
We assume the environment is a Partially Observable Markov Decision Process (POMDP) defined by:
$$\mathcal P = (\mathcal S, \mathcal A, \mathcal O, T, \Omega, R, \gamma)$$

where $s_t \in \mathcal S$ is the hidden state, $o_t \in \mathcal O$ is the observation, $a_t \in \mathcal A$ is the action, $T$ is the transition dynamics, $\Omega$ is the emission model, $R$ is the reward, and $\gamma \in [0, 1)$ is the discount factor.

### 2.2. Stage V0: Belief Verification
We project observations into a latent representation $z_t = E_\theta(o_t)$ and update a causal belief $h_t$ recursively using a gated recurrent model:
$$h_t = f_\theta(z_t, h_{t-1})$$

During training, we build a temporally verified belief using future context:
$$\tilde{h}_t = g_\theta(h_{t-k : t+k})$$

The representation discrepancy $d_t$ measures belief instability:
$$d_t = \|h_t - \tilde{h}_t\|_2$$

The consistency loss penalizes divergence from the smoother target:
$$\mathcal{L}_{cons} = \|h_t - \text{detach}(\tilde{h}_t)\|^2$$

### 2.3. Stage V1: Trust-Controlled Bellman operator
We map the discrepancy $d_t$ to belief reliability $c_t = \exp(-\lambda_{rel} d_t^2)$ and apply a floor constraint to protect against bootstrap collapse:
$$c_t' = c_{min} + (1 - c_{min}) c_t$$

The modified Bellman update gates bootstrap value propagation based on reliability:
$$y_t = r_t + \gamma c_t' \max_a Q(h_{t+1}, a)$$

Since $0 < c_t' \le 1$, the operator maintains contractive properties ($\gamma c_t' < 1$).

### 2.4. Stage V2: Adaptive Memory Modulation
V2 introduces a long-term memory state $M_t$ updated at an adaptive rate $\alpha_t$:
$$M_{t+1} = (1 - \alpha_t) M_t + \alpha_t h_{t-mod}$$
The representation is modulated dynamically using FiLM parameters $(\gamma_t, \beta_t)$ generated by a meta-controller based on TD error and latent gap:
$$h_t' = \gamma_t h_t + \beta_t$$

## 3. Project Structure

The codebase is split into reusable components, runners, and validation checks:

* **`src/ch_dqn/`**: Core implementation files.
  * [model.py](file:///src/ch_dqn/model.py): Causal filters, temporal smoothers, meta-controllers, and baseline networks.
  * [losses.py](file:///src/ch_dqn/losses.py): Consistency objectives, reliability computations, and modified Bellman operators.
  * [trainer.py](file:///src/ch_dqn/trainer.py): Training loops, sequence sampling, and SGD/Second-Order optimization step mapping.
  * [trainer_rl.py](file:///src/ch_dqn/trainer_rl.py): RL training harness for CartPole.
  * [replay.py](file:///src/ch_dqn/replay.py): Sequence replay buffer with subsequence sampling.
  * [envs/](file:///src/ch_dqn/envs/): Gymnasium POMDP wrappers with gaussian, uniform, exponential, correlated, and mixed noise models.
* **`experiments/`**: Script benchmarking and publication plotting utilities.
  * [main_benchmark.py](file:///experiments/main_benchmark.py): Runs standard sweeps across all noise configurations.
  * [plotting/](file:///experiments/plotting/): Publication quality plot generation.
* **`tests/`**: Integration and unit testing suite.
  * [test_replay_and_optimizer.py](file:///tests/test_replay_and_optimizer.py): Unit tests for replay buffers and custom optimizers.
  * [test_v1_v2_validation.py](file:///tests/test_v1_v2_validation.py): Validation suite verifying noise spikes response and relative model ranking.
  * [test_v1_v2_behavior.py](file:///tests/test_v1_v2_behavior.py): Behavioral metrics checking (TD variance decay, stability, and recovery).

## 4. Execution & Usage

### 4.1. Running Unit & Integration Tests
Verify mathematical and pipeline sanity:
```powershell
python -m pytest
```

### 4.2. Running the Learning Proof (Gaussian)
Train V2 on Gaussian noise and generate moving-average reward, TD variance, and alpha/reliability curves:
```powershell
$env:PYTHONPATH="src"
python run_learning_proof.py
```
This writes progress to `results/learning_proof_v2.csv` and saves visualizations to `output/figures/learning_proof_results.png`.

### 4.3. Benchmarking
To run targeted behavior sweeps across models (DQN, DRQN, R2D2, V0, V1, V2) and generate CSV data:
```powershell
$env:PYTHONPATH="src"
python run_targeted.py
```

To run the complete benchmark suite:
```powershell
$env:PYTHONPATH="src"
python -m experiments.main_benchmark
```

To regenerate publication plots from results:
```powershell
$env:PYTHONPATH="src"
python generate_all_plots.py
```
